# VHC Service Advisor View Specification
## Viewing & Managing Completed Health Checks

---

# 1. OVERVIEW

When a technician completes a health check, Service Advisors need to:
1. Review all findings grouped by severity
2. Add pricing (parts, labour, or total)
3. Add follow-up dates for advisory work
4. Mark potential MOT failures
5. Send to customer
6. Track authorisations
7. Mark work as complete
8. Close the health check

# 13. TYRE DISPLAY FORMAT

## 13.1 Calculation: Remaining Legal Tread

```typescript
// UK legal minimum = 1.6mm
const LEGAL_MINIMUM_MM = 1.6;

function calculateRemainingLegalTread(depths: { outer: number; middle: number; inner: number }) {
  const lowestReading = Math.min(depths.outer, depths.middle, depths.inner);
  const remaining = lowestReading - LEGAL_MINIMUM_MM;
  return Math.max(0, remaining);  // Don't show negative
}

// Example:
// depths = { outer: 3.5, middle: 3.8, inner: 3.2 }
// lowestReading = 3.2
// remaining = 3.2 - 1.6 = 1.6mm
```

# 19. API ENDPOINTS

```
# Repair Item Pricing
PATCH /api/v1/repair-items/:id
{
  "parts_price": 85.00,
  "labour_price": 25.00,
  // OR
  "price": 110.00,
  
  "is_mot_failure": true,
  "follow_up_date": "2026-06-01"
}

# Mark Work Complete
POST /api/v1/repair-items/:id/complete
Response: { "completed_at": "2026-01-15T14:30:00Z" }

# Unmark Work Complete
DELETE /api/v1/repair-items/:id/complete

# Close Health Check
POST /api/v1/health-checks/:id/close
Response: { "closed_at": "2026-01-15T15:00:00Z" }

# Validation error if work not complete:
{
  "error": "WORK_NOT_COMPLETE",
  "message": "Cannot close: 1 authorised item not marked as complete",
  "items": ["Front Brake Pads"]
}

# Generate PDF
GET /api/v1/health-checks/:id/pdf
Response: PDF file download
```
